# firmware 目录说明

本目录用于存放 `STM32F407` 固件工程和相关软件模块。

## 模块划分

- `Core`：系统入口与核心启动流程
- `BSP`：板级支持包，负责底层硬件驱动
- `App`：业务流程与任务调度
- `Algo`：数据处理与雨量识别算法
- `Comm`：通信协议与数据收发

## 使用原则

- 和寄存器、外设初始化强相关的代码优先放到 `BSP`
- 和“项目流程”相关的逻辑优先放到 `App`
- 和“数据怎么算”相关的逻辑优先放到 `Algo`
- 和串口、协议、数据打包相关的逻辑优先放到 `Comm`
- `Core` 尽量保持精简，只负责初始化和主流程入口

## 当前建议对应关系

- `main.c`、`stm32f4xx_it.c`：偏向 `Core`
- `My_ADC_DMA.c`、`My_USART.c`、`My_Led.c`：偏向 `BSP`
- 采样流程控制：后续可放到 `App`
- 特征提取：后续可放到 `Algo`
- 串口帧格式：后续可放到 `Comm`
